# Brand Setup Validation Checklist

Use this checklist after generating `brand/BRAND.md`, `brand/VOICE.md`, and `brand/VISUAL.md` to verify the setup is complete, specific, and actionable. A brand setup that passes all checks can immediately power other content skills. One that fails needs more conversation.

---

## Completeness Checks

These verify that all required sections are present and filled in.

### BRAND.md

- [ ] Brand name is set (not placeholder text)
- [ ] "What We Do" section describes the product/service concretely (not just a category)
- [ ] Elevator pitch exists and is under 3 sentences
- [ ] At least 1 competitor is named with strengths and weaknesses
- [ ] Differentiator is stated explicitly
- [ ] One-word essence is defined
- [ ] Primary audience section is fully populated (role, experience, company type, pain point)
- [ ] At least 1 publishing platform is listed with a frequency goal
- [ ] At least 3 content pillars are defined

### VOICE.md

- [ ] At least 3 core voice attributes are defined with positions and rationale
- [ ] Formality base level is set (1-10 scale)
- [ ] At least 1 per-context formality adjustment exists (e.g., blog vs. social)
- [ ] Humor style is defined (even if "none")
- [ ] At least 3 "sounds like us" example sentences exist
- [ ] At least 3 "doesn't sound like us" example sentences exist with reasons
- [ ] At least 3 preferred word substitutions exist
- [ ] At least 3 banned words/phrases exist
- [ ] Customer terminology is defined (what do we call our users?)

### VISUAL.md

- [ ] Primary color is defined (hex code preferred)
- [ ] At least 1 secondary or accent color exists
- [ ] Body font is specified (name or category preference)
- [ ] Heading font is specified (name or category preference)
- [ ] Photography/illustration style preference is captured
- [ ] At least 1 visual principle is articulated

---

## Specificity Checks

These verify that the content is precise enough to be useful. Vague brand guides produce vague content.

### Identity

- [ ] **Differentiator test:** Could a competitor copy-paste the differentiator into their own materials and have it be equally true? If yes, it is not specific enough.
- [ ] **Audience test:** Could you write a job posting targeting this exact audience? If the audience description is too vague to write a job post for, it needs more detail.
- [ ] **Elevator pitch test:** Does the pitch contain at least one concrete detail (a number, a specific outcome, a named problem)? "We help businesses grow" fails. "We cut deploy times from hours to minutes for teams shipping Kubernetes workloads" passes.

### Voice

- [ ] **Example sentence test:** Could a new writer read only the "sounds like us" / "doesn't sound like us" sentences and produce content in the right voice? If the examples are too generic to guide real writing, they need to be rewritten.
- [ ] **Banned word test:** For each banned word, is it clear why it is banned? "Leverage" is a common ban, but without context a writer might not know that "use" is the preferred alternative.
- [ ] **Formality test:** Pick a random topic. Could you write the first paragraph at the specified formality level without second-guessing? If the formality description is ambiguous, add clarifying examples.

### Visual

- [ ] **Color test:** Could a designer use the palette without asking follow-up questions? At minimum, a primary color and a text color should have hex values.
- [ ] **Typography test:** Could a designer set type for a blog post without guessing? At minimum, the heading and body font families should be specified (even if just "sans-serif, clean, modern").

---

## Actionability Checks

These verify that the brand guide can drive real content decisions, not just sit in a file.

### Content Generation

- [ ] **Blog post test:** Given the brand guide, could a content skill generate a blog post outline that clearly reflects the brand voice and targets the right audience? Try it. If the output feels generic, the guide is too vague.
- [ ] **Social post test:** Given the brand guide, could a content skill write a tweet/post that sounds like the brand and not like every other company? If it could belong to anyone, the voice section needs more edge.
- [ ] **Headline test:** Could you write 5 headlines that follow the voice guidelines? If all 5 sound the same, the voice lacks enough dimension.

### Decision Making

- [ ] **On-brand/off-brand test:** Pick a hypothetical content idea. Using only the brand guide, can you confidently say whether it is on-brand or off-brand? If you are unsure, the guide needs clearer boundaries.
- [ ] **Platform test:** Does the platforms section make it clear what type of content goes where? A writer should not have to guess whether a topic becomes a blog post or a Twitter thread.
- [ ] **Audience test:** If two pieces of content target different audience segments, does the guide help a writer adjust tone and depth appropriately?

## Common Failures

These are the most frequent problems with brand setups and how to fix them.

| Problem | Symptom | Fix |
|---|---|---|
| Generic differentiator | "We provide quality solutions" | Ask "could a competitor say this?" Repeat until the answer is no. |
| Vague audience | "Business professionals" | Push for title, seniority, industry, specific pain point. |
| No voice edge | Example sentences sound like any company | Add more "doesn't sound like us" examples. Define what makes the voice distinctive. |
| Aspirational vs. real | Visual guide describes a brand that doesn't exist yet | Label aspirational sections clearly. Document what exists today AND what they're aiming for. |
| Missing frequency | Platforms listed but no cadence | Force a commitment even if it's "1x/month." No frequency = no accountability. |
| Placeholder content | "TBD" or "TODO" in final output | Never ship placeholders. Either get the answer or remove the section with a note to revisit. |
| Too long | 50-page brand guide nobody reads | Keep BRAND.md under 200 lines. Push detail into VOICE.md and VISUAL.md. If those are also long, you're over-documenting. |
| No examples | Voice described abstractly with no samples | Every voice attribute needs at least one concrete example sentence. |

---

## Validation Process

1. Run through all Completeness checks. Fix any gaps.
2. Run through all Specificity checks. Strengthen any vague areas.
3. Run through all Actionability checks. If any test fails, the guide needs revision.
4. Check for Common Failures. Address any matches.
5. Present the validation results to the user with specific recommendations for improvement.

A brand setup is ready to use when it passes all checks without caveats. If compromises were made (e.g., no visual identity exists yet), document them explicitly so other content skills know to ask or skip those sections.
